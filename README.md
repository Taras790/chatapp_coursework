# Chat Application (COMP1549 — Task 1)

A multithreaded group chat application built in Java using Sockets and Swing GUI.
Supports coordinator election, private messaging, periodic pings, and fault tolerance.

---

## Project Structure

```
chatapp_coursework/
├── module-info.java                  # Java module declaration
├── coursework/
│   ├── Protocol.java                 # All protocol string constants
│   ├── ClientInfo.java               # Member data (ID, IP, port, join time)
│   ├── MessageRecord.java            # Timestamped message log entry
│   ├── MessageLogger.java            # Singleton — thread-safe message log
│   ├── ServerState.java              # Singleton + Observer subject — all server state
│   ├── GroupEventListener.java       # Observer interface for membership events
│   ├── ChatCommand.java              # Command pattern interface
│   ├── BroadcastCommand.java         # Command — send to all members
│   ├── PrivateMessageCommand.java    # Command — send to one member
│   ├── ListMembersCommand.java       # Command — request member list
│   ├── PongCommand.java              # Command — acknowledge ping
│   ├── QuitCommand.java              # Command — disconnect
│   ├── CommandFactory.java           # Parses input into Command objects
│   ├── PingScheduler.java            # Sends PING every 20 seconds
│   ├── ChatServer.java               # Server entry point
│   ├── ChatClient.java               # Client GUI entry point
│   └── ChatServerTest.java           # JUnit 5 tests (22 tests)
├── lib/                              # JUnit 5 jars (for running tests)
├── out/                              # Compiled .class files (generated by javac)
└── README.md
```

## How to Build and Run

> **All commands are run from the project root folder:** `chatapp_coursework/`

---

### Step 1 — Compile

```bash
javac -d out module-info.java coursework/*.java
```

This compiles everything into the `out/` folder. Run this once, or again whenever you change any `.java` file.

> To exclude the test file (no JUnit needed):
> ```bash
> javac -d out module-info.java $(ls coursework/*.java | grep -v ChatServerTest.java)
> ```

---

### Step 2 — Start the Server

Open a terminal and run:

```bash
java --module-path out -m coursework/coursework.ChatServer
```

**You must keep this terminal open.** The server does not stop unless you close it or press `Ctrl+C`.

You should see:
```
Chat server running on port 50000 ...
```

---

### Step 3 — Start a Client

Open a **new terminal** (leave the server running) and run:

```bash
java --module-path out -m coursework/coursework.ChatClient localhost
```

A GUI window will appear. Enter your unique ID when prompted.

> To connect from a **different machine**, replace `localhost` with the server's IP address:
> ```bash
> java --module-path out -m coursework/coursework.ChatClient 192.168.1.10
> ```

> To connect on a **custom port**:
> ```bash
> java --module-path out -m coursework/coursework.ChatClient localhost 50001
> ```

---

### Step 4 — Connect More Clients

Open additional terminals and repeat Step 3 for each client. Each must choose a **unique ID**.

## Using the Chat

| Action | How |
|---|---|
| Send a broadcast message | Type in the input box and press **Enter** or click **Send** |
| Send a private message | Type `@targetId your message` and press **Enter** |
| View all members | Click **List Members** or type `/list` |
| Leave the chat | Click **Quit** or close the window |

---

## Coordinator

- The **first client to connect** automatically becomes the **coordinator**.
- The coordinator's name is shown in **green** with `[COORDINATOR]` in the title bar.
- All other clients are told who the coordinator is when they join.
- If the coordinator disconnects (normally or abnormally), the **next oldest member** is automatically promoted.
- The new coordinator is notified immediately and their status bar updates.

## How It Works

### Protocol

| Server → Client | Meaning |
|---|---|
| `SUBMITNAME` | Request a unique ID from the client |
| `NAMEACCEPTED <id>` | ID accepted — client is now registered |
| `COORDINATOR_YOU` | You are the group coordinator |
| `COORDINATOR_IS <id>` | The current coordinator is this member |
| `COORDINATOR_CHANGED <id>` | A new coordinator has been elected |
| `MESSAGE <timestamp> <id>: <text>` | Broadcast message |
| `PRIVATE <timestamp> <id>: <text>` | Private message received |
| `MEMBER_LIST <data>` | Full member list response |
| `MEMBER_JOINED <id> ip:port` | A new member has joined |
| `MEMBER_LEFT <id>` | A member has left |
| `PING` | Server health check — client must reply with `PONG` |

| Client → Server | Meaning |
|---|---|
| `<id>` | Response to `SUBMITNAME` |
| `BROADCAST <text>` | Send message to everyone |
| `PRIVMSG <targetId> <text>` | Send private message |
| `LIST` | Request full member list |
| `PONG` | Response to `PING` |
| `QUIT` | Disconnect gracefully |

### Design Patterns Used

| Pattern | Where |
|---|---|
| **Singleton** | `ServerState`, `MessageLogger` — one shared instance across all threads |
| **Observer** | `GroupEventListener` interface + `ServerState` (subject) + `ChatServer` (observer) |
| **Command** | `ChatCommand` interface + `BroadcastCommand`, `PrivateMessageCommand`, etc. |

### Fault Tolerance

- The server sends a `PING` to all clients every **20 seconds**.
- Clients that do not reply with `PONG` within 5 seconds are removed automatically.
- If the **coordinator** disconnects (normal or abnormal), the server elects the **longest-connected remaining member** as the new coordinator.
- All remaining clients continue communicating without interruption.

## Running Tests

Tests require JUnit 5 jars (already in `lib/`).

**Compile tests:**
```bash
javac --module-path "out;lib/junit-jupiter-api_5.14.1.jar;lib/junit-platform-commons_1.14.1.jar" --add-reads coursework=org.junit.jupiter.api --add-modules org.junit.jupiter.api -d out coursework/ChatServerTest.java
```

**Run tests (in VS Code):** Open `ChatServerTest.java` and click the green run button next to any test method, or use the Testing panel in the sidebar.
