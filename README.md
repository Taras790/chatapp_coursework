# Chat Application

A multithreaded client-server chat application built in Java using Sockets and Swing GUI.

---

## Project Structure

```
chatapp_coursework/
├── module-info.java          # Java module declaration
├── coursework/
│   ├── ChatServer.java       # Server — accepts and manages client connections
│   └── ChatClient.java       # Client — Swing GUI for sending and receiving messages
└── out/                      # Compiled .class files (generated by javac)
```

---

## How It Works

### Protocol

The server and client communicate over TCP sockets using a simple text-based protocol:

| Message (Server → Client) | Meaning |
|---------------------------|---------|
| `SUBMITNAME`              | Server is requesting a screen name from the client |
| `NAMEACCEPTED <name>`     | The chosen name was accepted; client can now chat |
| `MESSAGE <text>`          | A broadcast message to display in the chat area |

| Message (Client → Server) | Meaning |
|---------------------------|---------|
| `<screen name>`           | Response to a `SUBMITNAME` request |
| `<any text>`              | Chat message to be broadcast to all clients |
| `/quit`                   | Disconnect from the server |

### Server (`ChatServer.java`)

- Listens on port **50000**.
- Uses a thread pool of up to 500 threads — each connected client gets its own `Handler` thread.
- Maintains two shared sets:
  - `names` — tracks taken screen names to prevent duplicates.
  - `writers` — holds a `PrintWriter` for every connected client, used to broadcast messages.
- When a client connects:
  1. Repeatedly sends `SUBMITNAME` until the client provides a unique, non-empty name.
  2. Sends `NAMEACCEPTED <name>` to confirm.
  3. Broadcasts `MESSAGE <name> has joined` to all existing clients.
  4. Forwards every subsequent message as `MESSAGE <name>: <text>` to all clients.
  5. On disconnect, broadcasts `MESSAGE <name> has left` and cleans up the shared sets.

### Client (`ChatClient.java`)

- Connects to the server via a TCP socket.
- Presents a Swing GUI with:
  - A **text field** at the bottom for typing messages (disabled until the name is accepted).
  - A **scrollable text area** showing the full chat history.
- On startup, responds to `SUBMITNAME` with a name entered via a dialog box.
- Once `NAMEACCEPTED` is received, the text field becomes editable and the window title updates to `Chatter - <name>`.
- Pressing **Enter** sends the typed message to the server and clears the field.

## How to Build and Run

### 1. Compile

From the project root (`chatapp_coursework/`):

```bash
javac -d out module-info.java coursework/ChatClient.java coursework/ChatServer.java
```

### 2. Start the Server

```bash
java --module-path out -m coursework/coursework.ChatServer
```

You should see: `The chat server is running...`

### 3. Start a Client

```bash
java --module-path out -m coursework/coursework.ChatClient localhost
```

Replace `localhost` with the server's IP address if running on a different machine.

Open multiple terminals and repeat this command to connect additional clients.

### 4. Disconnect

Type `/quit` in the message field and press Enter to leave the chat.
